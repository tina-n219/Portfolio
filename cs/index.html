Easy Animator

Model-View-Controller Architecture

We will build an application that helps to create simple but effective 2D animations from shapes (such as the above examples). One of our main goals will be to create a separation between how an algorithm or phenomenon is described and how it is drawn on the screen. This separation will allow us to create an application that will draw an animation according to how it is described in text, without being tied to how the description was produced.


The command-line arguments will be of the form
<-if name-of-animation-file>
<-iv type-of-view>
<-o where-output-show-go>
<-speed integer-ticks-per-second>


Four views for your Easy Animator application:

Textual

This view will show a textual description of the animation.


SVG

In this view, you will produce a textual description of the animation that is compliant with the SVG file format. The output of the program is saved in a file, and can be directly viewed in a web browser.


Visual

Implemented with JavaSwing


Hybrid

This view will have capabilities of the visual view, in that it will actually show the animation being played. It will also have capabilities of the SVG view, in that it will be able to create and export the animation in SVG format.
